# AgentBattery 用量获取使用说明

本说明只介绍 AgentBattery 的余额、今日用量和本月用量获取。

> 安全原则：API Key、Cookie、Authorization / Bearer Token 只能填在“安全变量”中。不要发送到聊天、截图、名称、URL 或数据处理输入框中。

---

## 一、基础用法：直接获取余额

适合：服务商提供标准余额接口，或你只想先确认余额可以获取。

### 1. 进入供应商编辑页

```text
主页 → 管理服务商 → 选择服务商 → 编辑
```

确认：

```text
服务商已启用
Base URL 正确
API Key 已在安全变量中保存
```

### 2. 使用通用网页账单

进入：

```text
通用网页账单 → 余额 Balance
```

有两种填写方式。

#### 方式 A：填写已有余额接口

填写或导入：

```text
请求方法：GET / POST
URL 模板：余额接口地址
Headers / Body：接口需要的内容
```

然后在该“余额 Balance”卡片内向下滚，找到：

```text
数据处理
```

填写服务商 Response 中余额字段的路径。

例如 Response：

```json
{
  "data": {
    "balance": 361.33
  }
}
```

填写：

```text
data.balance
```

若 Response 为：

```json
{
  "balance_usd": 94.38
}
```

填写：

```text
balance_usd
```

最后点击编辑窗口底部的：

```text
保存
```

回到主页刷新服务商。

#### 方式 B：导入浏览器 cURL（推荐）

适合网页控制台没有公开余额接口、但浏览器网页可以正常显示余额。

1. 登录服务商控制台。
2. 按 `F12` → 打开 `Network`。
3. 刷新余额页面。
4. 找到返回 JSON 的请求（可以通过搜索金额数字找到）。
5. 右键请求：`Copy` → `Copy as cURL (bash)`。
6. 在 AgentBattery 的“余额 Balance”中粘贴 cURL。
7. 点击：

```text
解析 cURL → 应用导入模板
```

8. 在同一张“余额 Balance”卡片继续向下滚，在“数据处理”填写 Response 的余额字段路径。
9. 在“安全变量”填写解析出来的变量，例如 `API_KEY`、`COOKIE` 或 `AUTHORIZATION_TOKEN`。
10. 保存并刷新。

> 只点“解析 cURL”不够，必须再点“应用导入模板”。

## 二、如何理解“数据处理”

它的作用是告诉 AgentBattery：

```text
服务商已经返回了一段 JSON，哪个数字应该显示为余额/用量？
```

它不是填写 API Key，也不是填写 Cookie。

例如 Response：

```json
{
  "data": {
    "account": {
      "remaining": 50
    }
  }
}
```

数据处理填写：

```text
data.account.remaining
```

还可以计算多个字段：

```text
sum(data.items[*].cost) / 1000000
```

允许：

```text
JSON 路径
sum(...)
数字
+ - * /
括号
```

不允许 JavaScript、Python 或任意代码。

## 三、今日与本月用量

“今日 Today”和“本月 Month”与余额配置方法相同，但分别有独立的请求和数据处理。

```text
今日 Today：填写服务商今日统计的请求与字段
本月 Month：填写服务商本月统计的请求与字段
```

如果服务商 Response 是：

```json
{
  "data": {
    "today_cost": 2.50,
    "month_cost": 112.35
  }
}
```

填写：

```text
今日 Today：data.today_cost
本月 Month：data.month_cost
```

不要把“余额请求”直接复制成“今日/本月请求”，除非 Response 本身已经同时给出正确的今日和本月字段。

### 时间参数注意事项

有些服务商的用量请求包含日期或时间戳。若 cURL 中有固定日期/时间戳，例如：

```text
startTs=1781884800
```

它可能只统计一个固定起点之后的累计值，不等于“今天”或“本月”。

此时应：

1. 在网页控制台分别切换“今日”和“本月”；
2. 从 Network 分别复制真实请求；
3. 分别导入 AgentBattery 的 Today 与 Month 卡片；
4. 再填写各自的数据处理。

## 四、请求失败时显示

余额、今日和本月可以分别选择失败后的显示策略。

### 隐藏

请求失败后不显示该项数值。

```text
适合：余额等实时性较高的数据。
```

### 显示缓存值

请求失败后继续显示上一次成功获取的数字。

```text
适合：接口偶尔不稳定，但你仍需要参考数值。
```

主页会提示“网页账单更新失败，显示缓存”。缓存不是本次实时结果。

### 显示估算值

适合“今日”和“本月”。当日/月统计请求失败但余额成功刷新时，AgentBattery 用余额减少量估算消耗。

```text
上次余额：100
当前余额：92
估算消耗：8
```

这是估算，不是服务商官方账单。

推荐：

```text
余额 Balance：隐藏
今日 Today：显示估算值
本月 Month：显示估算值
```

## 五、高级用法：安全变量

从 cURL 导入后，AgentBattery 会将敏感内容替换为变量。

常见变量：

```text
API_KEY
AUTHORIZATION_TOKEN
COOKIE
```

填写位置：

```text
通用网页账单 → 安全变量
```

规则：

- `API_KEY`：填 API Key；
- `AUTHORIZATION_TOKEN`：若模板中是 `Authorization: Bearer ${AUTHORIZATION_TOKEN}`，只填写 `Bearer ` 后面的令牌值；
- `COOKIE`：填浏览器请求 Header 中完整 Cookie 值；
- 已显示掩码时，说明已安全保存；不输入新值就不会替换。

变量值保存到 Windows 安全存储，不会写入普通配置 JSON。

## 六、常见提示与排查

### “未配置网页账单请求”

通常表示余额规则不完整，而不是网络无法连接。

检查：

```text
1. 是否已经“应用导入模板”？
2. 余额 Balance 是否有请求模板？
3. 余额 Balance 的“数据处理”是否为空？
4. 请求所需安全变量是否已经填写？
```

最常见原因：`数据处理`为空。

### HTTP 401 / 403

通常表示 API Key、网页 Bearer Token 或 Cookie 过期。

处理：

```text
重新在浏览器 Network 获取最新请求
重新导入或更新对应安全变量
```

### 请求成功但数值为空

通常表示“数据处理”路径不正确。

处理：

```text
打开该请求的 Network → Response
找到目标数值所在层级
按 JSON 结构填写路径
```

### 无法确认字段路径

可以截取 Response 中字段名和非敏感数字发来询问；务必不要包含：

```text
Cookie
Authorization
Bearer Token
API Key
用户邮箱、账号 ID
```
